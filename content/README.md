## 概要

このシステムは以下の課題を解決するために構築されています。

- **散在するナレッジの統合** — ドキュメント・手順書・調査メモを一か所に集約
- **情報の発見しやすさ** — 全文検索・セマンティック検索・タグによる多角的な探索
- **AI によるナレッジ活用** — 蓄積した情報を AI が読んで回答・要約・校正

## 主な機能

### ページ管理

| 機能 | 説明 |
|---|---|
| Markdown エディタ | GFM (GitHub Flavored Markdown) 完全対応。テーブル・コードブロック・チェックリストが使用可能 |
| Wiki リンク | `[[ページタイトル]]` 記法でページ間をリンク。存在しないページも作成誘導 |
| タグ管理 | 任意のタグを付与。AI によるタグ自動提案にも対応 |
| リビジョン履歴 | すべての編集履歴を保存。差分ビューアで変更点を確認可能 |
| いいね | ページに `♡` を付けてブックマーク代わりに活用 |
| 閲覧数 | ページごとの閲覧数を自動計測。ランキング表示に反映 |

### 検索

| 機能 | 説明 |
|---|---|
| 全文検索 | タイトル・本文を横断的に検索 |
| セマンティック検索 | Azure AI Search のベクトル検索で意味的に近いページを発見 |
| タグ検索 | タグ一覧ページから絞り込み |
| コマンドパレット | `Ctrl+K` からキーワード検索してページへ直接ジャンプ |

### AI 機能

| 機能 | 説明 |
|---|---|
| AI チャット (RAG) | Wiki の内容を文脈として参照しながら質問に回答。Claude Sonnet 使用 |
| AI 要約 | ページの内容を自動要約して右サイドバーに表示 |
| AI 校正 | 文章の誤字・改善点を指摘。Claude Opus 使用 |
| AI 補完 | 書きかけのテキストを続けて生成 |
| AI 翻訳 | 選択範囲を他言語に翻訳 |
| タグ自動提案 | 本文の内容からタグを AI が提案 |

### ファイル添付

PDF・Word・Excel・PowerPoint・画像ファイルをアップロードすると、GPT Vision が内容を解析して Markdown に変換します。

## クイックスタート

### 1. ページを作成する

1. 左サイドバーの **+ New Page** をクリック（または `Ctrl+K` → `new`）
2. タイトルを入力してエディタで内容を記述
3. **Save** をクリックして公開

### 2. Wiki リンクを張る

ページ内で他のページへリンクするには `[[ ]]` 記法を使います。

```markdown
[[ページタイトル]]
```

タイトルが完全一致するページへのリンクが自動生成されます。
存在しないタイトルを書いた場合も、クリック時に新規作成ページへ誘導されます。

### 3. 検索する

- `Ctrl+K` でコマンドパレットを開いてキーワードを入力
- ナビの **Search** から詳細検索
- タグ一覧から絞り込み

### 4. AI に質問する

- ページ下部の **AI Quick Ask** に質問を入力 → チャット画面へ遷移
- ナビの **Chat** から詳細な会話形式で質問
- ページ編集中は **AI アシスト** ツールバーから補完・翻訳・校正

## Markdown 記法リファレンス

### 基本構文

```markdown
# 見出し 1
## 見出し 2
### 見出し 3

**太字** / *斜体* / ~~取り消し線~~

[リンクテキスト](https://example.com)
[[Wiki内ページ名]]

- 箇条書き
  - ネスト

1. 番号付きリスト

> 引用文

`インラインコード`
```

### テーブル

```markdown
| ヘッダー1 | ヘッダー2 |
|---|---|
| セル | セル |
```

### チェックリスト

```markdown
- [x] 完了済みタスク
- [ ] 未完了タスク
```

### コードブロック（シンタックスハイライト対応）

言語名を指定するとシンタックスハイライトが適用されます（`typescript`・`python`・`sql`・`bash` など）。
